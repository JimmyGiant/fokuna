# Accessibility Checklist (WCAG 2.2 AA target)

Use for central flows before launch:

- [ ] Keyboard-only path for Auth, Tasks, Calendar move alternative, Focus, Journal, Settings
- [ ] Visible `:focus-visible` rings use `--fk-focus-ring` only
- [ ] Icon-only controls have accessible names
- [ ] Form errors are associated with inputs
- [ ] Modals trap focus and restore on close
- [ ] Drag interactions have a non-pointer alternative
- [ ] `prefers-reduced-motion` disables decorative motion
- [ ] Contrast checked for Light theme productive surfaces
